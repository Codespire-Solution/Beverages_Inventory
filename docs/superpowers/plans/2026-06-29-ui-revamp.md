# Beverage Inventory UI Revamp Implementation Plan

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** Re-skin the entire beverage-ERP UI in the drinkanothr.com-derived "Editorial" brand system (Mockup A), with no functional changes.

**Architecture:** Token-first. Define brand tokens + fonts once (`tailwind.config.js`, `globals.css`, `next/font`), restyle the shared components in `src/components/common/`, then sweep pages. Because pages reuse the common components, most of the visual change propagates from the foundation and component tasks.

**Tech Stack:** Next.js 14 (App Router), React 18, TypeScript, Tailwind CSS 3, `@phosphor-icons/react`, `next/font` (Playfair Display, Assistant, IBM Plex Mono), Recharts.

## Global Constraints

- Visual only. Do NOT change API routes, Prisma schema, auth logic, route slugs, hook signatures, or form field `name`s.
- Light theme only. No dark mode.
- Tokens (hex): `bg #FCFBF8`, `paper #FFFFFF`, `wash #F6F1E8`, `ink #17150F`, `ink-60 #6B655A`, `accent #E08A2B`, `accent-ink #B0640C` (accent TEXT only), `line #ECE6DA`, `mint #CFE6D0`, `berry #F4C9D4`, `litchi #F7D9BD`, `warn-bg #FBEAD3`, `warn-ink #9A5A12`, `ok-bg #D8EBD9`, `ok-ink #2F6135`.
- Accent orange is a FILL only; for orange text use `accent-ink` (WCAG AA).
- Radius: buttons/badges full-pill (`rounded-full`), cards `rounded-2xl` (16px), inputs `rounded-xl` (12px).
- Fonts: `font-serif` = Playfair Display (headings/numbers), `font-sans` = Assistant (body), `font-mono` = IBM Plex Mono (labels/badges/numbers, uppercase tracked).
- Zero em-dashes / en-dashes in any visible copy. Use hyphen, comma, or period.
- Icons: `@phosphor-icons/react` only. No emoji in UI. No hand-rolled SVG icons.
- Motion: animate transform/opacity only; ease `cubic-bezier(0.22,1,0.36,1)`; honor `prefers-reduced-motion`.
- Mockups of record: `mockups/dashboard-a-editorial-cream.html` (dashboard), `mockups/a5-login-splash.html` (login).
- Workspace is NOT a git repo. "Checkpoint" steps verify build/lint/visual instead of committing. Do not run `git`.
- Per-task verify command baseline: `npm run build` must succeed and `npm run lint` must report no new errors. Dev server: `npm run dev` (serves on APP_URL port 3002).
